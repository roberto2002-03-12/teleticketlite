## How project structure has to be:
```
.
└── teleticket/
    └── src/
        └── main/
            ├── java/
            │   └── com/
            │       └── app/
            │           └── teleticket/
            │               ├── users/
            │               │   ├── config/
            │               │   │   └── UsuarioConfig.java
            │               │   ├── controller/
            │               │   │   └── UsuarioResource.java
            │               │   ├── dto/
            │               │   │   ├── UsuarioCreateDTO.java
            │               │   │   └── UsuarioResponseDTO.java
            │               │   ├── entity/
            │               │   │   └── UsuarioEntity.java
            │               │   ├── exception/
            │               │   │   └── UsuarioException.java
            │               │   ├── repository/
            │               │   │   └── UsuarioRepository.java
            │               │   ├── service/
            │               │   │   ├── impl/
            │               │   │   │   └── UsuarioServiceImpl.java
            │               │   │   └── UsuarioService.java
            │               │   └── utils/
            │               │       └── UsuarioMapper.java
            │               ├── auth
            │               ├── events
            │               └── ...
            └── resources/
                └── application.properties
```
the project is organized in a modular way, each module represents a specific domain of the application, for example, user, events, qr, etc. Each module contains the following packages:
- config: contains the configuration classes for the module. (it's optional unless is needed)
- controller: contains the REST controllers that handle the HTTP requests and responses. You must use dto requests and must have a validator.
- dto: contains the Data Transfer Objects (DTOs) that are used to transfer data. It'll be used to work inside and outside the application, for example, in the controller and service layers.
- entity: contains the JPA entities that represent the database tables.
- exception: contains the custom exceptions that are used in the application.
- repository: contains the Spring Data JPA repositories that are used to interact with the database.
- service: contains the service interfaces and their implementations that contain the business logic of the application.
- utils: contains the utility classes that are used in one the module.

there are some main modules such as "config" that contains the main configuration of the application, for example, database configuration, AWS configuration, etc. and "utils" that contains the utility classes that are used in one or more modules, add them if you need them.
